Once Prometheus is installed, you can start issuing natural-language commands immediately — no configuration required beyond an AI backend. This guide walks you through one-shot mode, the interactive session, and the built-in commands you'll use most often.

## Your first command

One-shot mode lets you pass a single query as a quoted argument. Prometheus translates it, confirms if the command looks safe, and executes it.

```bash
prom "list my files"
```

Prometheus sends the query to your configured AI backend, receives a shell command (in this case, `ls -la`), runs a safety check, then executes it and streams the output back to your terminal.

Try a few more one-shot queries to get a feel for the translation:

```bash
prom "show disk usage"
prom "find all python files"
prom "what's my IP address"
prom "show system information"
```

<Tip>
  Use quotes around your query so your shell does not try to interpret special characters. Single quotes work too: `prom 'find files larger than 100MB'`.
</Tip>

## Interactive mode

For multi-step tasks, start an interactive session by running `prom` with no arguments. Prometheus keeps conversation context between queries, so you can refer back to previous results with natural language.

```bash
prom
```

You will see the Prometheus banner and a `>` prompt. Type queries just as you would in one-shot mode, but without quotes:

```
> list my files
> create a file called notes.txt with "Hello World"
> show contents of notes.txt
> find all python files in this directory
> show me disk usage
```

Because Prometheus maintains context across queries, follow-up requests like "add a line to it" or "show it again" resolve correctly to the file or command from the previous turn.

### Dry-run mode

Before running an unfamiliar command, enable dry-run mode to preview what Prometheus would execute without actually running it:

```
> dry-run on
> delete old_backup.tar.gz
$ rm old_backup.tar.gz
Dry-run mode: Command not executed
> dry-run off
```

## Special commands overview

These built-in commands are handled directly by Prometheus without going through the AI — they give you control over the session, configuration, and history.

<AccordionGroup>
  <Accordion title="help">
    Display the full list of available commands, flags, and quick actions.

    ```
    > help
    ```
  </Accordion>
  <Accordion title="config">
    Show the current configuration or change individual settings. Prometheus stores config in `~/.prometheus/config.json`.

    ```
    > config
    > config set timeout_seconds 30
    > config set auto_confirm_safe true
    ```

    Common config keys:

    | Key | Default | Description |
    |---|---|---|
    | `timeout_seconds` | `300` | Seconds before a command times out |
    | `default_model` | `llama3` | Ollama model name |
    | `auto_confirm_safe` | `false` | Skip confirmation for safe commands |
    | `history_size` | `100` | Number of commands to retain |

    To reset all settings to defaults, use the CLI subcommand: `prom config reset`.
  </Accordion>
  <Accordion title="history">
    Show, replay, or clear previous commands.

    ```
    > history          # Show all history
    > history 5        # Show last 5 entries
    > !!               # Repeat the last command
    > !3               # Repeat command at index 3
    > !git             # Repeat the last git command
    > clear-history    # Clear all history
    ```
  </Accordion>
  <Accordion title="exit / quit">
    End the interactive session.

    ```
    > exit
    ```

    You can also use `quit` or `q`.
  </Accordion>
</AccordionGroup>

## Safety in action

Prometheus warns you before running commands that are potentially destructive. If it detects patterns like `rm -rf`, it will pause and ask for explicit confirmation:

```
> delete all log files
⚠ DANGER: This command contains 'rm -rf' which could be destructive!
Are you SURE you want to run this? [y/N]:
```

Type `y` to proceed or press **Enter** to cancel. You can also use `dry-run on` at any time to preview commands before committing to them.

<Warning>
  Confirmation prompts are a safety net, not a guarantee. Always review the generated command before confirming, especially for file deletion or system-modification operations.
</Warning>
